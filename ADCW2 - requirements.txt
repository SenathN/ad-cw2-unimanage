


CS6004ES –Application Development
Coursework 2 (2025/26)

 The objective is to develop and document a web application using ASP.NET MVC with the C# programming language in Visual Studio. You must submit a comprehensive individual report detailing your implementation process and demonstrate the application in a Viva voce upon completion. It will be assessed using Visual Studio 2017 or any higher version and any features not working in the standard installation of Visual Studio 2017 or any higher version will not be assessed.
Submission Deadline:  2025 December 24th

Case Study: UniManage – University Course Management System (UCMS)

UniManage is a digital solution aimed at helping universities efficiently manage academic activities. The university wants to develop a comprehensive web application to allow students, lecturers, and administrators to manage courses, enrollments, assignments, and reporting. The platform should provide different functionalities for each user type while maintaining a secure and robust system.

Requirements for the System

User Registration and Login
•	Implement a secure registration and login system for Students, Lecturers, and Administrators.
•	Role-based access control to ensure users only see functions relevant to their role.

Dashboard
•	Students: View enrolled courses, assignment deadlines, and grades.
•	Lecturers: View teaching assignments, student submissions, and grading summaries.
•	Administrators: Overview of courses, enrollment statistics, and system usage.

Course Management
•	Administrators can create, update, and delete courses.
•	Lecturers can upload course materials and assignments.

Enrollment System
•	Students can browse available courses and enroll.
•	Validation of prerequisites and enrollment limits.

Assignment & Grading
•	Lecturers can create assignments, set deadlines, and record grades.
•	Students can submit assignments online and view feedback.

Reporting and Analytics
•	Administrators and lecturers can generate reports such as course popularity, student performance, and workload analysis.

Communication Module 
•	Messaging system between students and lecturers.

Security and Data Protection
•	Protect sensitive data such as grades and personal information.
•	Implement authentication, input validation, and protection against common web attacks.
